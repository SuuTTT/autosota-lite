# AgentMonitor: Research & Idea Management

The AgentMonitor manages the "cognitive state" of the project through two primary artifacts: the `idea_library.md` and the `research_report.md`.

## 1. Idea Library (`idea_library.md`)
The `idea_library.md` is a structured database of proposed, in-progress, and executed experiments. Each entry follows a specific template to ensure reproducibility and scientific tracking.

### Idea Template
```markdown
### IDEA-{ID}: {Concise Title}
- **Type**: [CODE | PARAM | LEAP]
- **Priority**: [HIGH | MEDIUM | LOW]
- **Risk**: [HIGH | MEDIUM | LOW]
- **Description**: Detailed explanation of the proposed change. What file is modified? What logic is added?
- **Hypothesis**: Why will this improve the metric? (e.g., "Scaling predicted distances by 0.79 compensates for Gemini 2.5's systematic underestimation in close-range samples.")
- **Status**: [PENDING | IN-PROGRESS | SUCCESS | FAILED | REJECTED]
- **Result**: Quantitative delta (e.g., "+0.012 reward") and qualitative observation.
```

### Classification (The LEAP Rule)
- **PARAM**: Hyperparameter tuning (e.g., changing a learning rate or a threshold).
- **CODE**: Engineering improvements (e.g., better regex for parsing, handling edge cases).
- **LEAP**: Fundamental methodological shifts (e.g., switching from raw model output to a blended multi-source estimator, adding a new modality/source like Key-Frames).
- **Deadlock Prevention**: If the monitor detects 3 consecutive `PARAM` ideas with marginal gains, it forces the next idea to be a `LEAP`.

## 2. Research Report (`research_report.md`)
The `research_report.md` serves as a deep-research grounding for the agents. It is typically generated by a research subagent (e.g., o4-mini-deep-research) and contains:

- **Follow-up Works**: Summaries of recent papers (2023-2025+) related to the task.
- **SOTA Techniques**: Proven best practices (e.g., "Chain-of-Thought prompting for spatial reasoning", "Ensembling via majority vote").
- **Root Cause Analysis**: Analysis of baseline failures (e.g., "Model systematically fails on allocentric distance due to coordinate frame confusion").
- **Actionable Ideas**: A list of optimization ideas derived from literature to seed the `idea_library.md`.

## 3. Monitor Trace (Traceback.md)
The monitor also maintains a real-time trace of tool outputs to identify:
- **Local Debugging Loops**: When an agent repeats the same failing fix 3+ times.
- **Environment Stagnation**: When dependencies are constantly missing or conflicting.
- **Intervention Protocol**: The Monitor intervenes with high-level guidance when a loop is detected.
